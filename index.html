CoMoSpeech: Speech Synthesis via Consistency Model


Abstract

Although denoising diffusion probabilistic models (DDPMs) have shown promising performance for speech synthesis, their practical application is largely limited by low inference speed, which usually involves a large number of iterative sampling steps. Recent advance in the consistency model has shown that it is possible to generate the high-quality image with only one-step sampling, by imposing a consistency constraint on the sampling trajectory of the ordinary differential trajectory. In this paper, we propose a \textbf{Co}nsistency \textbf{Mo}del-based {Speech} synthesis method, CoMoSpeech, which makes the consistency model applicable for speech synthesis tasks by designing specific and detailed pipelines of acoustic modelling. Our experiments show that the CoMoSpeech is able to produce both speaking and singing voices by performing only one sampling step, achieving an inference speed about 50 times faster than real-time on a single NVIDIA A100 GPU, which is close to the inference speed of FastSpeech2, making diffusion-sampling based speech synthesis truly practical. Both objective and subjective evaluations demonstrate that the CoMoSpeech can achieve fast inference speed similar to non-iterative methods, and the audio quality is also comparable to other multi-step diffusion model baselines.

The below audio samples show the comparison between Recordings, ground-truth (GT) mel + Vocoder, FastSpeech 2, DiffGAN-TTS, DiffSpeech, ProDiff, GradTTS, ResGrad-4 (Our method), and ResGrad-50 (Our method). Three samples are randomly selected from the test dataset of LJSpeech, LibriTTS, and VCTK. Details are illustrated as follows:

  • The number of inference steps used by diffusion models is shown behind the name of models. For example, ResGrad-4 stands for ResGrad with 4 inference steps. But FastSpeech 2 is a model name instead of denoting the number of inference steps.
  • For each dataset, the inference speed measured by the real-time factor (RTF) is shown in the brackets of the first case.
  • In order to improve the inference speed at each sampling step, ResGrad uses the same U-Net architecture in GradTTS but with twice fewer channels to reduce the model size (2.0M in ResGrad vs 7.6M in GradTTS).
